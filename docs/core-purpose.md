# Core Purpose — media2rag

## The Problem

ChatGPT, Claude, и другие AI-агенты **не работают с твоими данными правильно**:

1. **Файлы = сырой текст в контекст.** Нет подготовки, структурирования, извлечения ключевой информации.
2. **Нет памяти между сессиями.** Каждый чат начинается с нуля.
3. **Ответы из обучения, не из твоих файлов.** AI "вспоминает" то, на чём обучался, а не то, что ты загрузил.
4. **Контекстное окно ограничено.** Нельзя загрузить 50 книг и работать со всеми.
5. **Нет цитирования источников.** Непонятно, откуда AI взял информацию — из твоих данных или выдумал.

## The Solution

**media2rag — это система подготовки данных для RAG.**

```
Книга / Видео / Аудио / URL / Заметки
              │
              ▼
        CTG Pipeline
   (чистит, структурирует, извлекает)
              │
              ▼
         Qdrant + SQLite
    (вечная память, поиск по смыслу)
              │
              ▼
          RAG-чат
 (ответы на основе ТВОИХ данных)
```

### Ключевое отличие

> **Качество ответа AI зависит от качества подготовки данных.** Большинство систем пропускают этот шаг.

ChatGPT с файлами → сырой текст → средний ответ.
media2rag → подготовленные данные → точный ответ с источниками.

## What We Build First

**Core Engine — ядро для личного использования:**

| Компонент | Что делает |
|-----------|------------|
| `process` | Загрузил URL/файл → получил RAG-ready документ |
| `chat` | Задал вопрос → получил ответ с источниками из своих данных |
| `memory` | Система помнит контекст между сессиями |

**Это всё.** Остальное (скиллы, маркетплейс, B2B) — потом.

## Real Use Cases (Personal)

### 1. Книги и обучение
- Загрузил 20 книг по бизнесу → чат знает все концепции
- "Как автор X решает проблему Y?" → ответ с цитатой и ссылкой на главу
- Память между сессиями: "мы вчера обсуждали Z, продолжи"

### 2. Видео и курсы
- YouTube видео → транскрипт → структурированный контент
- "Что он говорил про HNSW?" → точный ответ с таймкодом
- Не нужно пересматривать 2-часовое видео

### 3. Заметки и мысли
- Свои заметки → searchable knowledge base
- "Что я думал про X?" → находит свои записи
- Связи между заметками через RAG

### 4. Документы и исследования
- PDF статьи → извлечённые claims, mental models, key terms
- "Какие аргументы за и против?" → структурированный ответ
- Источники всегда указаны

## Success Criteria (Core Engine)

- [ ] Загрузил 50+ документов разных форматов
- [ ] Задал вопрос → получил точный ответ с источником
- [ ] Начал новый чат → система помнит контекст
- [ ] Ответ основан на загруженных данных, не на обучении
- [ ] Можно сослаться на конкретный фрагмент документа
